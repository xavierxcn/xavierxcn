# Proposal: improve-decoration-budget-ux

## Summary

优化装修预算助手的交互体验，包括改进添加项目流程、增加全屏模式、并通过 frontend-design 技能提升页面美观度。

## Motivation

当前交互存在以下问题：
1. **添加项目流程不直观**：点击"添加项目"后直接插入空行，用户需要在行内的 datalist 中选择，体验割裂
2. **删除操作无确认**：误删后数据无法恢复
3. **无全屏模式**：复杂表格编辑时屏幕空间利用不足
4. **移动端删除按钮难发现**：依赖 hover 状态显示删除按钮
5. **分类折叠状态不持久化**：刷新页面后折叠状态重置

## Proposed Solution

### 1. 添加项目弹窗
- 点击"添加项目"按钮后弹出模态框
- 模态框展示该分类的所有建议项目，支持多选
- 支持输入自定义项目名称
- 确认后批量添加选中的项目

### 2. 删除确认机制
- 删除项目时显示简洁的确认提示
- 使用 CSS 动画实现平滑的删除过渡

### 3. 全屏模式
- 添加全屏切换按钮
- 全屏时隐藏页面其他元素，最大化工作区
- 支持 ESC 键退出全屏

### 4. 移动端优化
- 删除按钮始终可见（使用图标缩小占用空间）
- 触摸友好的按钮尺寸

### 5. 状态持久化
- 将分类折叠状态保存到 localStorage

### 6. 视觉美化（frontend-design）
- 统一配色和间距
- 优化模态框和按钮样式
- 添加适当的微交互动画

## Impact

- **用户体验**：更直观的添加流程，减少误操作
- **效率**：批量添加项目，全屏专注编辑
- **可用性**：移动端体验改善

## Scope

- 仅影响 `content/tools/decoration-budget/template.html`
- 不改变数据结构或计算逻辑
- localStorage 版本保持 v4（仅新增折叠状态字段）
