# Proposal: improve-astrology-ux

## Summary

改善占星板块的用户体验，解决两个核心问题：
1. 全局搜索无法搜索占星内容
2. 占星首页层级过深，用户需要多次点击才能到达具体内容

## Problem

### 搜索问题
当前搜索索引只包含博客文章（posts），占星内容（39个条目）完全不在搜索范围内。用户无法通过顶部搜索栏找到星座、行星等信息。

### 导航问题
当前占星首页设计：
- 顶部大面积 hero 区域占用过多空间
- 只展示 4 个分类卡片（星座/行星/宫位/相位）
- 用户需要：首页 → 分类页 → 条目页，共 3 层导航

## Solution

### 1. 扩展搜索索引
在 `generate_search_index` 中添加占星内容，使用户可以直接搜索"白羊座"、"太阳"等关键词。

### 2. 重新设计占星首页
采用紧凑型布局，在首页直接展示所有条目：
- 精简顶部区域，使用水平 tab 切换分类
- 在首页直接显示所有条目的网格视图
- 支持分类筛选，无需跳转即可浏览所有内容
- 保留搜索/筛选功能

## Scope

- **build**: 扩展搜索索引生成逻辑
- **render**: 重新设计占星首页模板

## Dependencies

无外部依赖

## Risk

低风险 - 主要是 UI 调整和索引扩展
